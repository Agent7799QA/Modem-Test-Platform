## [0.2.0] - 2026-07-01

### ✨ Добавлено
- **Интерактивное CLI меню** (`main_pyMenu.py`) с использованием Rich для красивого вывода
- **Управление всеми параметрами модема** через единое меню:
  - Частота (3500, 4000, 4500, 6500 МГц)
  - Режим работы (swarm+, swarm, longrange)
  - Скорость (rate: 5, 10, 25, 40, 50 Гц)
  - Протокол (crsf, sbus, mavlink, raw)
  - FHSS режим (0-4)
  - DSSS режим (0-7)
  - Адрес сети (PAN: 0-65534)
  - Адрес управления (BIND: 0-65534)
  - Переключение LED
  - Перезагрузка модема
- **Состояние сессии** (`SessionState`) — хранение всех параметров в течение сессии
- **Правая колонка в меню** — отображение текущих значений параметров
- **CRSF парсер** (`protocols/crsf_parser/`) — полный перенос из старого проекта
- **Мониторинг порта данных** (`monitoring/`):
  - `BaseMonitor` — базовый класс для мониторинга
  - `RxMonitor` — мониторинг и парсинг CRSF телеметрии
  - `StatCollector` — сбор статистики (min/max/avg) для LQ и RSSI
- **Эмуляция команд** (`emulation/CommandEmulator`) — отправка RC-каналов через CRSF протокол
- **Rich дашборд** (`ui/rich_dashboard.py`) — красивое отображение телеметрии в реальном времени
- **Команды телеметрии** в CLI:
  - `monitor` — мониторинг телеметрии в реальном времени
  - `collect` — сбор статистики по сигналам
  - `emulate` — эмуляция команд (RC-каналы)

### 🔧 Изменено
- **Рефакторинг `CrossfireAdapter`** — разделение ответственности:
  - `send_command()` — чистая отправка команд
  - `check_error()` — проверка ответа на ошибки
  - `verify_change()` — проверка изменения параметров через `print`
- **Удалены лишние задержки** в `verify_change()` и `_toggle_parameter()` (команды применяются мгновенно)
- **Переработана логика `CrossfireProtocol.send_command()`**:
  - Убрано автоматическое переподключение при пустом ответе
  - Добавлен список команд без данных (`COMMANDS_WITHOUT_DATA`)
  - Добавлен метод `reopen()` для переоткрытия порта после ребута
- **Переработана логика `reboot()`**:
  - Использует `"Initialization done!"` как признак готовности
  - Не закрывает порт вручную
- **Улучшена очистка ответов** в `SerialTransport.send_command()`:
  - Использует `".."` как единственный маркер окончания ответа
  - Корректно удаляет эхо команд
- **Исправлено сравнение строковых параметров** — приведение к нижнему регистру для `mode` и `protocol`

### 🐛 Исправлено
- Ошибка "порт занят" при выполнении команд после перезагрузки
- Ошибка "нет ответа" для команд без данных (`freq`, `mode`, `led`, и т.д.)
- Обрезание SN в `PrintParser` — удаление символов `>` и пробелов
- Парсинг `led_state` — добавлен в `PrintParser`
- Конфликт имен `send_command()` в `CrossfireAdapter` и `CrossfireProtocol`
- Импорт `time` в `config_commands.py`
- Ошибка при смене режима (`mode`) — сравнение с учетом регистра

### 🗑️ Удалено
- Удален `_reconnect()` из логики `send_command()` (переподключение только при явной необходимости)
- Удален `_send_command_and_verify()` — заменен на явные вызовы `send_command()` → `check_error()` → `verify_change()`

## [0.1.0] - 2026-06-30

### ✨ Добавлено
- **Базовая архитектура проекта**:
  - `transport/` — транспортный уровень (Serial, TCP, UDP)
  - `protocols/crossfire/` — текстовый протокол для команд модема
  - `devices/adapters/` — адаптеры устройств (BaseAdapter, CrossfireAdapter)
  - `devices/configuration.py` — модель конфигурации модема
- **Парсеры команд модема**:
  - `PrintParser` — парсинг ответа на `print`
  - `StatParser` — парсинг ответа на `stat`
  - `TtlStatParser` — парсинг ответа на `ttlstat`
- **CLI команды** для порта конфигурации:
  - `configure` — проверка подключения и чтение конфигурации
  - `read-config` — чтение конфигурации
  - `read-stat` — чтение состояния канала
  - `set-frequency` — установка частоты
  - `set-mode` — установка режима
  - `toggle-led` — переключение LED
  - `reboot` — перезагрузка модема
- **Обработка ошибок**:
  - Порт занят — понятное сообщение для пользователя
  - Порт не найден — понятное сообщение
  - Невалидные значения — `ValueError` с описанием

### 🔧 Изменено
- **Логика чтения ответов** в `SerialTransport.send_command()`:
  - Использование `\r\n` вместо `\n`
  - Чтение всех данных с таймаутом
  - Удаление эха команд

### 🐛 Исправлено
- Ошибка "порт занят" при повторном открытии
- Некорректное чтение ответов на `print` и `stat`
