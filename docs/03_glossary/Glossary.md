# Purpose

Документ определяет единую терминологию проекта.

Все документы, исходный код и пользовательская документация должны использовать термины исключительно в значении, определенном данным документом.

---

# Terms

## Device

Физическое устройство, участвующее в испытаниях.

Примеры:

- Crossfire TX Modem
- Crossfire RX Modem
- Flight Controller
- RC Transmitter
- Spectrum Analyzer

---

## Device Type

Тип устройства.

Например:

- Modem
- Flight Controller
- RC
- Jammer
- Analyzer

---

## Configuration

Набор параметров одного Device.

Configuration может быть:

- считана;
- изменена;
- сохранена;
- загружена;
- применена;
- сравнена.

Configuration принадлежит только одному Device.

---

## TestConfiguration

Полное описание условий выполнения одного этапа испытаний.

Включает:

- Configuration каждого Device;
- параметры выполнения испытания;
- критерии оценки (при необходимости).

---

## TestPlan

Описание полного сценария испытаний.

Состоит из последовательности Step.

---

## Step

Минимальное атомарное действие Test Engine.

Примеры:

- Connect Device
- Apply Configuration
- Wait
- Collect Measurement
- Verify Criteria
- Generate Report

---

## TestRun

Экземпляр выполнения TestPlan.

Содержит информацию о времени запуска, используемых устройствах, результатах выполнения и итоговом статусе.

---

## Measurement

Результат измерения, полученный во время выполнения TestRun.

Источником Measurement является устройство.

---

## Analysis

Обработка Measurement с целью вычисления показателей качества испытания.

---

## Metric

Вычисляемый показатель качества.

Примеры:

- LQ
- Packet Loss
- RSSI
- Reconnect Count

Список метрик расширяем.

---

## Criterion

Правило оценки результата испытания.

Пример:

LQ ≥ 97%.

---

## Verdict

Итоговое решение по результатам испытания.

Примеры:

- PASS
- FAIL
- ERROR
- CANCELLED

---

## Report

Документ, содержащий сведения о выполненном испытании.

Включает:

- информацию об устройствах;
- примененные Configuration;
- результаты Measurement;
- вычисленные Metric;
- Verdict;
- журнал выполнения.

---

## Test Engine

Ядро платформы.

Отвечает за выполнение TestPlan.

Не зависит от пользовательского интерфейса и конкретного типа устройств.

---

## Adapter

Компонент, реализующий взаимодействие Test Engine с конкретным типом Device.

---

## Protocol

Способ обмена данными между платформой и Device.

Пример:

- Crossfire
- MAVLink

---

## Session

Логическое соединение между платформой и Device на время выполнения операций.

---

## Workspace

Рабочая область пользователя.

Содержит конфигурации, TestPlan, результаты испытаний и отчеты.

---

# Naming Rules

Во всех документах и исходном коде используются только термины, определенные данным документом.

Добавление нового термина допускается только после обновления Glossary.
